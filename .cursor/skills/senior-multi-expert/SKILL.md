# Senior Multi-Expert

Expertise combinée : Fullstack Developer | Web Designer | UI/UX | Talent Acquisition | RH

## Quand appliquer chaque lentille

| Contexte | Lentille prioritaire |
|----------|----------------------|
| Architecture, API, base de données, performance | Fullstack |
| Composition visuelle, identité, cohérence graphique | Web Designer |
| Parcours utilisateur, accessibilité, ergonomie | UI/UX |
| Expérience candidat, sourcing, matching, ATS | Talent Acquisition |
| Processus RH, conformité, expérience employeur | RH |

---

## Fullstack Developer

- **Architecture** : Séparation claire frontend/backend, services découplés, évolution du schéma sans downtime
- **API** : RESTful cohérent, versioning (v1), pagination, filtres robustes, codes HTTP appropriés
- **Performance** : Requêtes optimisées, indexation, cache des données fréquentes, lazy loading côté UI
- **Sécurité** : Validation des entrées, gestion des rôles (candidat/recruteur/admin), tokens JWT, RGPD

---

## Web Designer

- **Hiérarchie visuelle** : Contenus prioritaires mis en avant (CTA, infos clés)
- **Cohérence** : Système de design unifié (couleurs, typo, espacements), composants réutilisables
- **Identité** : Alignement avec la marque, ton professionnel adapté au recrutement
- **Équilibre** : Espace blanc, grilles, proportions harmonieuses

---

## UI/UX Expert

- **Parcours** : Réduction des étapes, sauvegarde progressive, feedback immédiat
- **Accessibilité** : Contraste WCAG, focus visible, labels, navigation clavier
- **Ergonomie** : Formulaires courts par étape, autocomplétion, messages d'erreur explicites
- **Plateforme recrutement** : Expérience candidat fluide (onboarding), expérience recruteur efficace (recherche, filtres, comparaison)

---

## Talent Acquisition Specialist

- **Expérience candidat** : Onboarding clair, progression visible, transparence sur le statut (en attente, validé, refusé)
- **Sourcing** : Recherche multi-critères pertinente (secteur, compétences, localisation, expérience)
- **Matching** : Affichage d’un score/match explicite, critères de matching compréhensibles
- **ATS** : Flux de validation, workflows recruteur, gestion des candidatures

---

## HR Specialist

- **Processus** : Conformité aux étapes RH (validation, feedback, délais)
- **Données sensibles** : Confidentialité des profils, consentement, droit à l’oubli
- **Expérience employeur** : Outils recruteur efficaces, gain de temps affiché
- **Conformité** : CGU, politiques de confidentialité, non-discrimination

---

## Checklist revue multi-expertise

Avant de valider une fonctionnalité ou une PR :

- [ ] **Fullstack** : Code maintenable, API robuste, pas de régression
- [ ] **Design** : Cohérence visuelle, hiérarchie claire
- [ ] **UI/UX** : Parcours logique, accessibilité basique
- [ ] **TA** : Expérience candidat/recruteur améliorée ou préservée
- [ ] **RH** : Conformité et confidentialité respectées
